MobMenu

We struggled through a lot of projects with different ways of creating mobile menus that workt as we wanted. We decided to create a way of doing it, this is it.

This is an example, offcourse you can add an extra menu to the right with the same logic. And a tip if you want to use a fixed header in mobile is to locate the mobile-header in a relativ positioned container. Since the fixed header stopps working in Windows mobile if it's located in a fixed container.

Structure of html

<body>
	<div class="mobile-content">
		<div class="ui-block"></div>
		<header>
			<div class="desktop-header">
				<h1>Make your window smaller..</h1>
				<p>This demo is only presented when viewport is smaller then 678px</p>
			</div>
			<div class="mobile-header">
				<div id="mobile-nav-toggle"></div>
			</div>
		</header>
		<div class="page-container">
			{PAGE HTML}
		</div>
	</div>
	<div class="mobile-left">
		{MENU HTML}
	</div>
</body>

CSS

body.open-left{
	overflow: hidden;
}
	body.open-left .mobile-content{
		margin-left:75%;
		position: fixed;
	}
	body.open-left .mobile-left{
		overflow: auto;
		left:0px;
	}
	body.open-left .ui-block{
		position: absolute;
		left:0px;
		right:0px;
		top:0px;
		bottom:0px;
		z-index: 999999;
	}

/* LEFT SECTION (HIDDEN ON START) */
.mobile-left{
	position: fixed;
	overflow-y:auto;
	top:0px;
	width: 75%;
	height: 100%;
}
.mobile-left{
	left:-75%;
	transition: left 0.5s;
	background-color: #EEE;
}
.mobile-content{
	transition: margin-left 0.5s;
	width:100%;
	position: relative;
}

And the small JavaScript

document.querySelector('.ui-block').addEventListener('click', function (e) {
	e.stopPropagation();
	e.preventDefault();
	document.body.className = document.body.className.replace('open-left', '');
});
var menu_toggle_button = document.getElementById('mobile-nav-toggle');
menu_toggle_button.onclick = function() {
	document.querySelector('body').classList.add('open-left');
}
